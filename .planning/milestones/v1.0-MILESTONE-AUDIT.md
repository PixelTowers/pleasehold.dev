## Integration Issues

### 1. Worker missing API_URL in Docker (NOTF-07) — Warning

**File:** `docker-compose.yml`, worker service environment block
**Impact:** In production Docker deployment, verification emails link to `http://localhost:3001/verify/<token>` instead of the configured public API URL. Users clicking the link get a broken URL.
**Fix:** Add `API_URL: ${API_URL:-http://localhost:3001}` to the worker service environment block.
**Affected flow:** Double opt-in email verification in Docker production only.

### 2. pending_verification not filterable (DASH-01) — Info

**File:** `packages/trpc/src/routers/entry.ts` line 10
**Impact:** `entryStatusEnum` is `z.enum(['new', 'contacted', 'converted', 'archived'])` — excludes `pending_verification`. Entries in this state appear in unfiltered views but the status filter dropdown cannot isolate them.
**Fix:** Add `pending_verification` to the filter enum (but NOT to `updateStatus`/`bulkUpdateStatus` — that's a system-assigned status).
