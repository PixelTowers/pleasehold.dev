2. **Create `apps/api/Dockerfile`** -- Multi-stage build using turbo prune:

   **Stage 1 (base):** `FROM node:22-alpine AS base`
   - `RUN apk add --no-cache libc6-compat`
   - `ENV PNPM_HOME="/pnpm"` and `ENV PATH="$PNPM_HOME:$PATH"`
   - `RUN corepack enable && corepack prepare pnpm@10.30.1 --activate`

   **Stage 2 (pruner):** `FROM base AS pruner`
   - `RUN pnpm add -g turbo@2.4.4`
   - `WORKDIR /app`
   - `COPY . .`
   - `RUN turbo prune @pleasehold/api --docker`

   **Stage 3 (installer):** `FROM base AS installer`
   - `WORKDIR /app`
   - `COPY --from=pruner /app/out/json/ .`
   - `RUN pnpm install --frozen-lockfile`

   **Stage 4 (builder):** `FROM installer AS builder`
   - `COPY --from=pruner /app/out/full/ .`
   - `RUN pnpm turbo build --filter=@pleasehold/api...`

   **Stage 5 (runner):** `FROM base AS runner`
   - `WORKDIR /app`
   - `RUN addgroup --system --gid 1001 nodejs && adduser --system --uid 1001 appuser`
   - `COPY --from=builder /app/ .`
   - `USER appuser`
   - `EXPOSE 3001`
   - `CMD ["node", "apps/api/dist/index.js"]`

   File must start with a 2-line ABOUTME comment.

3. **Create `apps/worker/Dockerfile`** -- Same multi-stage pattern as API:
   - Identical stages 1-4 but prune/build for `@pleasehold/worker` instead of `@pleasehold/api`
   - Runner stage: `CMD ["node", "apps/worker/dist/index.js"]`
   - No EXPOSE (worker has no HTTP port)
   - File must start with a 2-line ABOUTME comment.

4. **Create `apps/web/Dockerfile`** -- Build with Node, serve with nginx:

   **Stage 1 (base):** Same as above (node:22-alpine, corepack, pnpm)

   **Stage 2 (pruner):** Same pattern, prune `@pleasehold/web`

   **Stage 3 (installer):** Same as API

   **Stage 4 (builder):** `FROM installer AS builder`
   - `COPY --from=pruner /app/out/full/ .`
   - `ARG VITE_API_URL=http://localhost:3001`
   - `ENV VITE_API_URL=$VITE_API_URL`
   - `RUN pnpm turbo build --filter=@pleasehold/web...`

   **Stage 5 (runner):** `FROM nginx:alpine AS runner`
   - `COPY --from=builder /app/apps/web/dist /usr/share/nginx/html`
   - `COPY apps/web/nginx.conf /etc/nginx/conf.d/default.conf`
   - `EXPOSE 80`
   - `CMD ["nginx", "-g", "daemon off;"]`

   IMPORTANT: Vite env vars are build-time only. The `VITE_API_URL` ARG is baked into the JS bundle during build. However, since we use nginx reverse proxy (see nginx.conf below), the web app does NOT need VITE_API_URL at all -- it uses relative URLs (/api/*, /trpc/*) which nginx proxies to the API container. So the ARG is a fallback but the nginx proxy approach is preferred.

   File must start with a 2-line ABOUTME comment.

5. **Create `apps/web/nginx.conf`**:
   ```nginx
   server {
       listen 80;
       server_name _;
       root /usr/share/nginx/html;
       index index.html;

       # Proxy API and tRPC requests to the api service
       location /api/ {
           proxy_pass http://api:3001;
           proxy_set_header Host $host;
           proxy_set_header X-Real-IP $remote_addr;
           proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
           proxy_set_header X-Forwarded-Proto $scheme;
       }

       location /trpc/ {
           proxy_pass http://api:3001;
           proxy_set_header Host $host;
           proxy_set_header X-Real-IP $remote_addr;
           proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
           proxy_set_header X-Forwarded-Proto $scheme;
       }

       # Proxy verification endpoint
       location /verify/ {
           proxy_pass http://api:3001;
           proxy_set_header Host $host;
           proxy_set_header X-Real-IP $remote_addr;
       }

       # Proxy docs endpoints
       location /doc {
           proxy_pass http://api:3001;
           proxy_set_header Host $host;
       }

       location /docs {
           proxy_pass http://api:3001;
           proxy_set_header Host $host;
       }

       # Cache hashed static assets aggressively
       location /assets/ {
           expires 1y;
           add_header Cache-Control "public, immutable";
       }

       # SPA fallback: serve index.html for client-side routes
       location / {
           try_files $uri $uri/ /index.html;
       }
   }
   ```
   This approach means the web container proxies all /api/*, /trpc/*, /verify/*, /doc, /docs to the api container, and serves the React SPA for everything else. The frontend uses relative URLs (already the case with the Vite proxy pattern in dev), so no build-time API URL is needed.

   File must start with a 2-line ABOUTME comment (use `#`).
