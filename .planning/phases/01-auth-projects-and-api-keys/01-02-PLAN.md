  <name>Task 1: Build project tRPC router with CRUD operations and field configuration</name>
  <files>

  <action>
    Create `packages/trpc/src/routers/project.ts` with the following procedures:

    **project.create** — protectedProcedure mutation
    - Input: `z.object({ name: z.string().min(1).max(100), mode: z.enum(['waitlist', 'demo-booking']) })`
    - Use `ctx.db.transaction()` to insert BOTH the project row AND the field config row atomically.
    - Project row: `{ userId: ctx.user.id, name: input.name, mode: input.mode }`.
    - Field config row: Set defaults based on mode:
      - `waitlist`: `{ collectName: false, collectCompany: false, collectMessage: false }` (email only)
      - `demo-booking`: `{ collectName: true, collectCompany: true, collectMessage: true }` (all fields)
    - Return the created project with its field config.
    - CRITICAL: The field config row MUST be created in the same transaction. If this is missed, Phase 2 field validation will break (Pitfall 3 from RESEARCH.md).

    **project.list** — protectedProcedure query
    - No input required.
    - Query: `ctx.db.query.projects.findMany({ where: eq(projects.userId, ctx.user.id), with: { fieldConfig: true }, orderBy: [desc(projects.createdAt)] })`.
    - Returns all projects owned by the current user with their field configs.

    **project.getById** — protectedProcedure query
    - Input: `z.object({ id: z.string().uuid() })`
    - Query with ownership guard: `where: and(eq(projects.id, input.id), eq(projects.userId, ctx.user.id))`.
    - Include `with: { fieldConfig: true }`.
    - Throw `TRPCError({ code: 'NOT_FOUND' })` if not found (also covers unauthorized access — don't distinguish).
    - CRITICAL: Every project query MUST include the userId check. Extract a `requireProjectOwnership` pattern or inline it consistently.

    **project.update** — protectedProcedure mutation
    - Input: `z.object({ id: z.string().uuid(), name: z.string().min(1).max(100) })`
    - NOTE: Input schema deliberately EXCLUDES `mode`. Mode is immutable after creation (Pitfall 2 from RESEARCH.md). Only `name` can be updated.
    - Verify ownership before update. Update `name` and `updatedAt`.
    - Return updated project.

    **project.updateFields** — protectedProcedure mutation
    - Input: `z.object({ projectId: z.string().uuid(), collectName: z.boolean(), collectCompany: z.boolean(), collectMessage: z.boolean() })`
    - Verify project ownership first (query projects table with userId check).
    - Update the `project_field_configs` row matching the projectId.
    - Update `updatedAt` on both the field config row and the parent project row.
    - Return the updated field config.

    **Update router.ts:**
    - Import `projectRouter` from `./routers/project`.
    - Merge into `appRouter`: `project: projectRouter`.
  </action>

  </verify>
