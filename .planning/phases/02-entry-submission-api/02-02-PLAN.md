ABOUTME comments, then export `apiKeyAuth` -- a function that takes `auth` (ReturnType of createAuth) and `db` (Database type) and returns a Hono middleware.

The middleware:
1. Extract API key from `x-api-key` header. If missing, return 401:
   ```json
   { "error": { "code": "MISSING_API_KEY", "message": "x-api-key header is required" } }
   ```

2. Call `auth.api.verifyApiKey({ body: { key: apiKey } })`. If result is not valid or has no metadata, return 401:
   ```json
   { "error": { "code": "INVALID_API_KEY", "message": "API key is invalid or revoked" } }
   ```

3. Extract `projectId` from `result.key.metadata.projectId`. If missing, return 401 (same error).

4. Load project with field config:
   ```typescript
   const project = await db.query.projects.findFirst({
     where: eq(projects.id, projectId),
     with: { fieldConfig: true },
   });
   ```
   If not found, return 404:
   ```json
   { "error": { "code": "PROJECT_NOT_FOUND", "message": "Project not found" } }
   ```

5. Set context variables for downstream handlers:
   - `c.set('project', project)` -- the full project object with fieldConfig
   - `c.set('apiKeyId', result.key.id)` -- for audit trail
   - `c.set('db', db)` -- database instance for the handler

6. Call `next()`.

Import types from Hono (`Context`, `Next`), drizzle-orm operators (`eq`), and schema (`projects`) from `@pleasehold/db`.

**IMPORTANT:** Do NOT use `verifyProjectKey` from `@pleasehold/auth` here. That utility requires a projectId parameter (it checks key belongs to a specific project), but in this middleware we're EXTRACTING the projectId FROM the key's metadata. The key IS the project selector. Use `auth.api.verifyApiKey` directly and read metadata.projectId.

**Type safety note:** The Hono context `c.set()` / `c.get()` typing requires declaring the context variables. Use Hono's type-safe variables pattern:
```typescript

  project: typeof projects.$inferSelect & { fieldConfig: typeof projectFieldConfigs.$inferSelect };
  apiKeyId: string;
  db: Database;
};
```
Export this type so the route handler and index.ts can use it for proper typing.

5. Attempt insert with atomic position assignment (ENTR-05):
   ```typescript
   const insertResult = await db
     .insert(entries)
     .values({

       email: data.email,
       name: data.name ?? null,
       company: data.company ?? null,
       message: data.message ?? null,
       metadata: data.metadata ?? null,
       position: sql`(SELECT COALESCE(MAX(${entries.position}), 0) + 1 FROM ${entries} WHERE ${entries.projectId} = ${project.id})`,
   ```
