<task type="auto">
  <name>Task 1: Build and verify Docker Compose stack from clean state</name>
  <files>.planning/phases/09-qa-verification/09-02-QA-REPORT.md</files>
  <action>
Test the complete Docker self-hosting experience from scratch. Record all results in a QA report.

**Step 1 — Prepare clean environment:**
1. Stop any running local dev services (API, web) to free ports 3001 and 8080
2. Clean up any existing Docker volumes to simulate fresh install:
```bash
docker compose down -v 2>/dev/null
docker compose down --volumes --remove-orphans 2>/dev/null
```
3. Verify ports 3001 and 8080 are free:
```bash
lsof -i :3001 -i :8080 2>/dev/null | grep LISTEN
```

**Step 2 — Create .env from example:**
1. Check if `.env` exists. If not, copy from `.env.example`:
```bash
cp .env.example .env
```
2. Set the required values in `.env`:
   - `POSTGRES_PASSWORD=qatest_secure_pw_2026`
   - `BETTER_AUTH_SECRET=qa-test-secret-at-least-32-characters-long-enough`
   - Leave SMTP_* empty (notifications are tested in Plan 09-03)
   - Keep default ports (API_PORT=3001, WEB_PORT=8080)

IMPORTANT: Do NOT overwrite an existing .env that has real configuration. If .env already exists, read it to check if the required values are set. Only create/modify if needed for testing.

**Step 3 — Build and start Docker Compose:**
```bash
docker compose up --build -d 2>&1 | tail -50
```
- Record: Build output for each service (success/failure)
- Record: Any build errors or warnings

**Step 4 — Wait for services and verify health:**
```bash
# Wait up to 120 seconds for all services to be healthy
timeout=120
elapsed=0
while [ $elapsed -lt $timeout ]; do
  docker compose ps --format json 2>/dev/null
  # Check if api and web are running
  api_status=$(docker compose ps api --format '{{.Status}}' 2>/dev/null)
  web_status=$(docker compose ps web --format '{{.Status}}' 2>/dev/null)
  if echo "$api_status" | grep -q "Up" && echo "$web_status" | grep -q "Up"; then
    echo "Services are up after ${elapsed}s"
    break
  fi
  sleep 5
  elapsed=$((elapsed + 5))
done
docker compose ps
```
- Verify: postgres, redis, api, worker, web are all running
- Verify: migrate service completed successfully (exited 0)
- Record: Service status table

**Step 5 — Verify API health endpoint:**
```bash
curl -s http://localhost:3001/health | jq .
```
- Verify: Returns `{ "status": "ok" }`

Also verify through nginx proxy (web port):
```bash
curl -s http://localhost:8080/health | jq .
```
- Verify: Same `{ "status": "ok" }` response

**Step 6 — Verify web dashboard is accessible:**
```bash
curl -s -o /dev/null -w "HTTP_STATUS:%{http_code}\nCONTENT_TYPE:%{content_type}\n" http://localhost:8080/
```
- Verify: HTTP 200, Content-Type contains text/html
- Verify: Response contains the React SPA shell (check for root div or JS bundle reference)

**Step 7 — Verify OpenAPI docs accessible:**
```bash
curl -s http://localhost:8080/docs -o /dev/null -w "HTTP_STATUS:%{http_code}\n"
curl -s http://localhost:8080/doc | jq .openapi,.info.title
```
- Verify: /docs returns 200 (Scalar UI)
- Verify: /doc returns JSON with openapi: "3.0.0" and title: "pleasehold API"

**Step 8 — Create user and project for functional test:**
```bash
# Sign up
SIGNUP=$(curl -s -c cookies.txt -X POST http://localhost:8080/api/auth/sign-up/email \
  -H "Content-Type: application/json" \
  -d '{"name":"Docker QA","email":"docker-qa@pleasehold.dev","password":"DockerTest123!"}')
echo "$SIGNUP" | jq .

# Create project
PROJECT=$(curl -s -b cookies.txt -X POST http://localhost:8080/trpc/project.create \
  -H "Content-Type: application/json" \
  -d '{"json":{"name":"Docker Test Project","mode":"waitlist"}}')
echo "$PROJECT" | jq .
PROJECT_ID=$(echo "$PROJECT" | jq -r '.result.data.json.id')

# Create API key
APIKEY=$(curl -s -b cookies.txt -X POST http://localhost:8080/trpc/apiKey.create \
  -H "Content-Type: application/json" \
  -d "{\"json\":{\"projectId\":\"$PROJECT_ID\",\"name\":\"Docker QA Key\"}}")
echo "$APIKEY" | jq .
API_KEY=$(echo "$APIKEY" | jq -r '.result.data.json.key')
```
- Verify: Signup returns user object
- Verify: Project creation returns project with id
- Verify: API key starts with `ph_live_`

**Step 9 — Submit entry via Docker-hosted API:**
```bash
curl -s -w "\nHTTP_STATUS:%{http_code}\n" -X POST http://localhost:8080/api/v1/entries \
  -H "Content-Type: application/json" \
  -H "x-api-key: $API_KEY" \
  -d '{"email":"docker-entry@example.com"}' | jq .
```
- Verify: Status 201, entry returned with id, email, position

**Step 10 — Check migration service logs:**
```bash
docker compose logs migrate 2>&1 | tail -20
```
- Verify: Migrations ran successfully (no errors in log output)
- Verify: "drizzle-kit push" or "migrate" completed
